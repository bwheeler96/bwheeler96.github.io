Startup Idea Every Day #14: Browser Compatibility Testing Platform

This is the 14th post in my series "A new startup idea every day".

As a developer, I'm constantly building. I personally hate stopping production to do things such as write tests, do browser compatibility. How great would it be if there was a company that provided a browser testing & deploy-time testing service?

Imagine a website like NewRelic, where you could sign up, and add a piece of code to your site that would do the following

  1. Crawl your site and report errors every time the server is restarted.
  2. Crawl your site using different user agents for browser compatibility
  3. Generate analytic reports on availability across devices & response times

Monetization of a product like this would be as simple as letting developers know it exists. Sites that aren't quite as useful as this are cropping up like gangbusters (New Relic, Honeybadger.io, Google Analyics, Clicky, and countless others.)

The hardest part of creating this Software as a Service would be getting the dev-power to replicate the exact behavior of the countless devices that you would need for generating the report. Also understanding the subtle nuances of each device's behavior, and staying current to the new devices that show up would prove challenging.
